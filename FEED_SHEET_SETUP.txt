STEP 3 — VERIFY BUDGET CELL REFERENCES
════════════════════════════════════════════════
After pasting, check col S shows the right numbers:

  S1 should be ~1,399  (current tickets sold)
  S2 should be ~2,000  (BEP ticket target)
  S3 should be ~2,850  (SO ticket target)
  S4 should be ~15.42  (TPD for sold out)
  S5 should be ~13.98  (TPD for BEP)
  S7 should be ~69.95% (BEP tickets %)
  S8 should be ~49.09% (SO tickets %)

⚠️ If any values are wrong, the cell references in the Dashboard tab
   may be different. Screenshot col S and send to Claude to fix.


════════════════════════════════════════════════
STEP 4 — PUBLISH THE FEED SHEET AS CSV
════════════════════════════════════════════════
1. File → Share → Publish to web
2. First dropdown: select "DB26APR" (the tab name)
3. Second dropdown: select "Comma-separated values (.csv)"
4. Click Publish
5. Copy the URL (ends with &output=csv)
6. Paste into the dashboard Settings panel


════════════════════════════════════════════════
STEP 5 — CONFIGURE THE DASHBOARD
════════════════════════════════════════════════
In the dashboard Settings panel, enter:
  Festival Name:     DB26APR
  CSV URL:           (the URL from step 4)
  Event Date:        23/04/2026
  Sale Launch Date:  07/12/2025


════════════════════════════════════════════════
FOR FUTURE FESTIVALS
════════════════════════════════════════════════
1. Add a new tab to the Feed Sheet (e.g. DB27XXX)
2. Paste the same IMPORTRANGE formulas pointing to the new source sheets
3. Publish the new tab as CSV
4. In the dashboard → Settings → "+ New Festival" → paste the new CSV URL
5. The festival tab will appear in the dashboard automatically
